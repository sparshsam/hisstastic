# Agent Notes

## Repository Role

Hiss-Tastic is a browser-first arcade game (v0.9.0) with a preserved Python/Pygame reference implementation. The web runtime (`web/`) is the primary target.

## Operating Rules

- Do not remove existing gameplay features without explicit request.
- Keep Python installer binaries out of the repository.
- Preserve local-first privacy posture; cloud features (Supabase high scores) are opt-in per save.
- Update README, architecture, changelog, and privacy notes when behavior changes.
- Follow the ecosystem standards at https://github.com/sparshsam/ecosystem-standards.

## Replay and Ghost Rules

- Preserve deterministic replay behavior when changing game state, input, spawn, timing, or scoring logic.
- Keep replay files local JSON artifacts unless explicitly modifying cloud features.
- Ghost replay code is visualization-only. Ghosts must never affect scoring, collisions, spawn placement, difficulty, or replay verification.

## Browser Runtime

- The web runtime (`web/`) is the primary build target. Python is a preserved reference.
- PWA behavior is limited to offline static asset caching unless explicitly approved.
- Do not add telemetry, accounts, third-party analytics, or multiplayer without explicit review.
- Supabase integration (cloud high scores) uses anonymous submissions — no auth required.

## Git Safety

- Never force-push `main`.
- Prefer normal `git push` after commits.
- The `android/` directory is gitignored (generated by Capacitor) — build from source.
- Keep `.vercel/` out of version control (environment-specific config).
